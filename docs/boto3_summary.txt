1. Creating the Boto3 QuickSight Client
   The file begins with:
       client = boto3.client("quicksight")

   This creates a Boto3 client that allows the code to interact with
   the AWS QuickSight API. All create/update operations on dashboards
   are executed through this client.

2. Creating a new analysis (create_analysis)
   The commented section in the file shows the following call:

       client.create_analysis(
           AwsAccountId = analysis_json["AwsAccountId"],
           AnalysisId   = analysis_json["AnalysisId"],
           Definition   = analysis_json["Definition"],
           Name         = analysis_json["Name"],
           ...
       )

   Purpose:
   - Creates a brand new QuickSight analysis (dashboard).
   - The required fields used in this project are:
       • AwsAccountId
       • AnalysisId
       • Name
       • Definition

   Where does “Definition” come from?
   It is generated through:
       analysis_json = analysis_1.compile()

   The compile() function aggregates:
       - datasets
       - sheets
       - visuals
       - parameters
       - filter groups
       - calculated fields
     into a single JSON structure that matches the Boto3 API format.

3. Updating an existing analysis (update_analysis)
   The second commented section shows how an update is performed.
   Although the sample mistakenly repeats create_analysis, the
   intended API call is:

       client.update_analysis(...)

   Purpose:
   - Updates an existing QuickSight analysis instead of creating a new one.
   - It uses the same fields as create_analysis, but typically does not
     require redefining permissions or tags.

4. describe_analysis_definition (not present but important)
   In QuickSight-as-code workflows, this call is frequently used to
   retrieve the full JSON definition of an existing analysis:

       client.describe_analysis_definition(
           AwsAccountId = ...,
           AnalysisId   = ...
       )

   This is useful for:
       - debugging generated definitions
       - extracting a template from an existing dashboard

5. Global workflow inside create_analysis_lambda.py
   The file demonstrates a complete end-to-end flow:
   1) Create an Analysis object (account ID, analysis ID, name)
   2) Build a Definition object (datasets, sheets, visuals, filters, parameters)
   3) Attach the Definition to the Analysis
   4) Compile everything into JSON with:
          analysis_json = analysis_1.compile()
   5) Send the JSON to QuickSight using:
          client.create_analysis(...)  or  client.update_analysis(...)

6. Conclusion
   The create_analysis_lambda.py file acts as a full example of how
   to programmatically build a QuickSight dashboard and deploy it
   using the AWS Boto3 QuickSight API. It shows the structure required
   by the API, the minimum parameters needed, and the final calls used
   to create or update an analysis.
